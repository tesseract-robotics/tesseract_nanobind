# Tesseract 0.33 → 0.34 Upgrade Design

## Version Changes

| Dependency | Current | Target |
|---|---|---|
| ros_industrial_cmake_boilerplate | 0.7.1 | 0.7.4 |
| tesseract | 0.33.1 | 0.34.1 |
| tesseract_planning | 0.33.1 | 0.34.0 |
| trajopt | 0.33.0 | 0.34.4 |
| ifopt | 2.1.3 | 2.1.4 |
| ruckig | 0.9.2 | 0.15.3 |

Unchanged: boost_plugin_loader 0.4.3, descartes_light 0.4.9, opw_kinematics 0.5.2.

## Critical API Changes

### 1. ifopt absorbed into trajopt_ifopt

The external `ethz-adrl/ifopt` library is replaced by a reimplementation inside `trajopt_ifopt/core/`. All `ifopt::` types move to `trajopt_ifopt::` with API changes:

- `ifopt::Bounds` → `trajopt_ifopt::Bounds` (new accessors: `getLower()`/`getUpper()` instead of `lower_`/`upper_` members, new `BoundsType` enum)
- `ifopt::Component` → `trajopt_ifopt::Component` (PascalCase → camelCase: `GetValues()` → `getValues()`)
- `ifopt::VariableSet` → `trajopt_ifopt::Variables` (redesigned, new `getHash()` pure virtual)
- `ifopt::ConstraintSet` → `trajopt_ifopt::ConstraintSet` (new `Differentiable` base, `FillJacobianBlock` → `update()` + `getJacobian()`)
- `ifopt::Problem` → `trajopt_ifopt::Problem` (new interface)
- `ifopt::Composite` → split into `CompositeVariables` / `CompositeDifferentiable`

New base class `Differentiable` with `Mode` enum (`kStackRows`, `kSumToScalar`), `update()`, `getCoefficients()`, `getJacobian()`.

### 2. Variable system redesign

- `trajopt_ifopt::JointPosition` (inherits `ifopt::VariableSet`) → `trajopt_ifopt::Var` (standalone)
- New: `Node` (contains multiple `Var` instances), `NodesVariables` (manages all nodes)
- All constraint constructors change from `shared_ptr<const JointPosition>` to `shared_ptr<const Var>`

### 3. Constraint constructor changes

**CartPosConstraint:** `CartPosInfo` struct → individual params (var, manip, source_frame, target_frame, offsets) + `RangeBoundHandling` enum.

**JointPosConstraint:** vector of JointPosition → single Var, added bounds-based constructor, added `RangeBoundHandling`.

**Collision evaluators:** `CollisionCache` parameter removed (now internal pooling).

### 4. Serialization: Boost → Cereal

- All boost serialization includes replaced by cereal
- New JSON format support (`toArchiveStringJSON`, `fromArchiveStringJSON`)
- Serialized data NOT backwards compatible
- Python-facing API stays similar (`to_xml`, `from_xml`, `to_file`, `from_file`)

### 5. Profile changes

- `getStaticKey()` removed from all profile classes
- `createKey()` static function added to Profile base
- Cereal serialization friend declarations instead of Boost

### 6. Enum naming (Google style)

- New `BoundsType` enum: `kUnbounded`, `kEquality`, `kLowerBound`, `kUpperBound`, `kRangeBound`
- New `RangeBoundHandling` enum: `kKeepAsIs`, `kSplitToTwoInequalities`
- Existing enums may have values renamed to `kCamelCase`

### 7. Ruckig 0.9 → 0.15

- `PositionExtrema` → `Bound`
- Transparent if wrapped by tesseract's time parameterization

## Strategy

Compiler-driven, bottom-up. Build C++ deps first, then fix binding files module by module guided by compilation errors. The compiler tells us exactly what broke.

## Phases

### Phase 1: Fetch & build C++ deps

1. `vcs import ws/src < dependencies.rosinstall` (fetches new tag sources)
2. `scripts/build_tesseract_cpp.sh` (builds upstream C++)
3. Validate clean C++ build before touching bindings

### Phase 2: Core module bindings

Fix compilation errors in low-level modules (likely minimal changes):
- `tesseract_common_bindings.cpp`
- `tesseract_geometry_bindings.cpp`
- `tesseract_scene_graph_bindings.cpp`
- `tesseract_state_solver_bindings.cpp`
- `tesseract_collision_bindings.cpp`
- `tesseract_environment_bindings.cpp`
- `tesseract_kinematics_bindings.cpp`
- `tesseract_urdf_bindings.cpp`
- `tesseract_srdf_bindings.cpp`

### Phase 3: Serialization (Boost → Cereal)

- Update `tesseract_serialization_bindings.cpp`
- Remove boost includes, update to cereal
- Keep Python API stable (`to_xml`, `from_xml`, etc.)

### Phase 4: ifopt → trajopt_ifopt merge (THE BIG ONE)

This is the largest single change. The separate `ifopt` Python module merges into `trajopt_ifopt`.

**Decision: Remove ifopt module entirely (Option B)**
- Delete `ifopt_bindings.cpp` and `ifopt` Python module
- All types live in `trajopt_ifopt` only — matches upstream C++ structure
- Breaking change for Python users (clean break)
- Update CMakeLists.txt to remove ifopt build target
- Update all internal imports referencing `tesseract_robotics.ifopt`

Changes needed:
- New `Var`/`Node`/`NodesVariables` bindings
- New `Differentiable` base class binding
- Rewrite `PyConstraintSet` trampoline (camelCase, new methods)
- Rewrite `PyVariableSet` → `PyVariables` trampoline
- New `Bounds` API (accessors instead of members)
- New enums: `BoundsType`, `RangeBoundHandling`, `Differentiable::Mode`

### Phase 5: Planner & SQP modules

- `tesseract_motion_planners_bindings.cpp` — likely minor
- `tesseract_motion_planners_ompl_bindings.cpp` — likely minor
- `tesseract_motion_planners_trajopt_bindings.cpp` — profile `getStaticKey` removal
- `tesseract_motion_planners_trajopt_ifopt_bindings.cpp` — profile updates
- `tesseract_motion_planners_descartes_bindings.cpp` — likely minor
- `tesseract_motion_planners_simple_bindings.cpp` — likely minor
- `trajopt_sqp_bindings.cpp` — `ifopt::Problem` → `trajopt_ifopt::Problem`, possible enum renames

### Phase 6: Task composer & Python layer

- `tesseract_task_composer_bindings.cpp` — minor profile/key changes
- Update `composer.py`, `program.py` if they reference changed types
- Regenerate `.pyi` stubs
- Run full test suite + all examples

### Phase 7: Documentation

- Update CLAUDE.md "Tesseract 0.33 API Changes" section for 0.34
- Update example status
- Note new API patterns (Var/Node, Differentiable, camelCase)

## Affected Files (23 binding files, ~6 heavy changes)

**Heavy rewrite:** ifopt_bindings.cpp, trajopt_ifopt_bindings.cpp, trajopt_sqp_bindings.cpp
**Moderate:** tesseract_serialization_bindings.cpp, tesseract_motion_planners_trajopt_bindings.cpp, tesseract_motion_planners_trajopt_ifopt_bindings.cpp
**Light/none:** remaining 17 binding files

## Risk Assessment

- **Phase 4 (ifopt merge)** is the highest risk — complete API redesign
- **Serialization** format change means saved data is not backwards compatible (document this)
- **Ruckig** jump is large (0.9→0.15) but should be transparent through tesseract's wrapper
- **Python backwards compatibility** depends on Phase 4 Option A vs B decision
